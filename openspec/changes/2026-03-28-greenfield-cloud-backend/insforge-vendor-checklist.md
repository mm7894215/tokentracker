# InsForge：上线前向官方/控制台确认的清单

公开文档与 Context7 索引**不足以**作为数据库灾备与合规承诺的依据。若最终选择 **InsForge 托管 Postgres**，请在合同、控制台或支持工单中**书面确认**下列条目（勾选后保留证据链接或截图存档）。

## 数据库与备份

- [ ] **自动备份频率**（例如每日 / 每小时）及是否包含 WAL 连续归档。
- [ ] **可恢复窗口**：最近一次成功备份可回滚的时间范围；是否提供 **PITR（按秒/按分钟）** 或等价能力。
- [ ] **备份保留期**：保留多少天的历史备份；升级档位后是否变化。
- [ ] **恢复流程**：自助恢复还是工单；典型 **RPO/RTO** 是否有说明。

## 区域与合规

- [ ] **数据驻留区域**（例如单一区域 / 可选区域）及创建项目后是否可迁移区域。
- [ ] **项目删除**：删除项目后数据与备份是否在供应商侧**立即不可恢复**（与多数云 BaaS 类似，但需确认条款）。

## 配额与运行时

- [ ] **Edge Functions**：并发、超时、体大小、冷启动相关限制（若文档未列明）。
- [ ] **Postgres**：连接数、存储上限、单表或单库大小告警策略。

## Auth（若使用 InsForge Auth）

- [ ] **OAuth 应用审核**（Google/GitHub）在生产环境的要求与回调域名配置流程。
- [ ] **JWT 格式与轮换**：`accessToken` 有效期、刷新策略、是否与标准 OIDC 对齐（便于 CLI/桌面端集成）。

---

**说明**：本清单仅降低选型风险，不构成法律意见；关键条款以你与 InsForge 的协议为准。
